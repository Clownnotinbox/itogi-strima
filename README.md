# Стрим-цитаты

Локальное приложение для записи стримов, расшифровки речи и отбора цитат с таймкодами.

Поддерживаются записи YouTube и Twitch, Twitch-каналы в прямом эфире, VK Video и прямые ссылки на видео или аудио. Результаты хранятся локально и при необходимости публикуются на GitHub Pages.

## Возможности

- запись Twitch-эфира частями без остановки на время расшифровки;
- автоматическое переподключение, если эфир кратковременно пропал;
- расшифровка через faster-whisper;
- предварительный отбор фраз правилами и повторная оценка через Ollama;
- таймкоды, поиск, копирование и экспорт в JSON или HTML;
- обновление цитат, итогов и публичной страницы каждые 2 часа;
- локальное хранение базы, аудио и расшифровок.

## Требования

- Windows 10 или 11;
- Python 3.11;
- [FFmpeg](https://ffmpeg.org/download.html) в `PATH`;
- [Ollama](https://ollama.com/);
- Git, если нужна публикация на GitHub Pages;
- не менее 16 ГБ оперативной памяти для модели Ollama по умолчанию.

Видеокарта NVIDIA не обязательна. Без CUDA расшифровка работает на процессоре, но медленнее.

## Установка

Клонируйте репозиторий и откройте папку проекта:

```powershell
git clone <URL-РЕПОЗИТОРИЯ>
cd <ПАПКА-РЕПОЗИТОРИЯ>
```

Создайте окружение и установите зависимости:

```powershell
py -3.11 -m venv .venv
.venv\Scripts\activate
python -m pip install --upgrade pip
pip install -r requirements.txt
```

Для NVIDIA CUDA установите дополнительные библиотеки:

```powershell
pip install -r requirements-cuda.txt
```

Установите модель Ollama:

```powershell
ollama pull gemma4:12b
```

## Запуск

Сначала запустите Ollama:

```powershell
ollama serve
```

Затем откройте `run_live.bat`. Интерфейс будет доступен по адресу:

```text
http://127.0.0.1:8001
```

Чтобы остановить приложение, закройте окно запуска или откройте `stop_live.bat`.

## Использование

1. Вставьте ссылку на запись или Twitch-канал. Несколько ссылок добавляются по одной на строку.
2. Нажмите **Начать обработку**.
3. Текущий этап появится в блоке **В работе**.
4. Готовые цитаты появятся ниже. Таймкод открывает исходное видео в нужном месте.
5. Параметры публичной страницы находятся в раскрываемом блоке **Публичная страница**.

Для Twitch-канала приложение остаётся запущенным после окончания эфира и ждёт следующий. Если соединение пропало ненадолго, запись возобновляется в рамках той же сессии. По умолчанию время ожидания — 15 минут.

## Где хранятся данные

Все рабочие данные находятся в папке `data`:

```text
data/
├── audio/          временные аудиофрагменты
├── transcripts/    расшифровки TXT и JSON
├── summaries/      промежуточные итоги
├── quote_exports/  экспорт цитат
└── *.sqlite3       локальная база
```

Папка `data` исключена из Git. Она создаётся автоматически при первом запуске. Для полной очистки остановите приложение и удалите содержимое этой папки.

`public/data.json` содержит данные публичной страницы. Не публикуйте этот файл, если цитаты не должны быть доступны в интернете.

## Настройка

Параметры задаются переменными окружения перед запуском. Основные значения:

| Переменная | По умолчанию | Назначение |
| --- | ---: | --- |
| `KRYL_PORT` | `8001` | порт локального интерфейса |
| `KRYL_WHISPER_MODEL` | `large-v3` | модель faster-whisper |
| `KRYL_WHISPER_DEVICE` | `auto` | `auto`, `cuda` или `cpu` |
| `KRYL_OLLAMA_MODEL` | `gemma4:12b` | модель Ollama |
| `KRYL_TOP_N` | `5` | число цитат для обычной записи |
| `KRYL_LIVE_QUOTE_LIMIT` | `30` | максимум live-цитат |
| `KRYL_LIVE_SUMMARY_SECONDS` | `7200` | интервал итогов в секундах |
| `KRYL_LIVE_RECONNECT_GRACE_SECONDS` | `900` | ожидание возврата эфира |
| `KRYL_PAGES_AUTO_PUSH` | `0` | автоматическая отправка GitHub Pages |

Пример для запуска на процессоре:

```powershell
$env:KRYL_WHISPER_DEVICE="cpu"
.\run_live.bat
```

Все параметры находятся в `config.py`.

## GitHub Pages

Публичная страница собирается в папке `public`. В корне репозитория находится перенаправление на неё, поэтому для Pages можно использовать ветку `main` и каталог `/root`.

1. Создайте пустой публичный репозиторий на GitHub.
2. Подключите его к проекту:

   ```powershell
   .\setup_pages_repo.bat https://github.com/USER/REPOSITORY.git
   ```

3. На GitHub откройте **Settings → Pages**.
4. Выберите **Deploy from a branch**, ветку `main` и каталог `/root`.
5. Для запуска с автоматическим обновлением используйте:

   ```powershell
   .\run_live_pages.bat
   ```

Страница и цитаты обновляются каждые 2 часа и после завершения эфира. Для разовой публикации откройте `deploy_pages.bat`.
